It's All Public

The trouble with the Detective class as we've built it so far is that all its members, both data variables and functions, are public in nature. The class will behave as though these members were all declared with the C++ keyword public, letting us access them directly via the dot operator.

Unfortunately, this means that any code anywhere in the program can access -- and more importantly, modify! -- this information in any way that the syntax of JavaScript will allow.

In a mystery story, it would be bad news if someone tampered with the evidence, altering the detective's notebook. The bad consequence in the detective story is that the wrong person gets blamed for the crime. The bad consequence in programming is not only that the wrong thing happens, but also that debugging becomes very difficult because the problem could be caused by code literally anywhere in the program. There's no good way to track down the "culprit."

To get to something resembling private members of a class, which cannot be directly accessed and altered from outside the class, we will need to introduce a new concept: closures.
