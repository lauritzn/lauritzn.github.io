Getters and Setters Using Closures

How did that go? We came up with this for our class, complete with getters and setters: 

   var Detective = function(dName, dSidekick){
     var name = dName;
     var sidekick = dSidekick;
     var notebook = []; 
     notebook.push("It was a dark and stormy night"); 
     return {
       getName: function() {	
         return name;
       },	
       getSidekick: function(){
         return sidekick;
       },		
       setName: function(newName) {
         name = newName;	
       },
       setSidekick: function (newSidekick){
         sidekick = newSidekick;
       },
       solveCrime: function(){ 
         console.log(name + " has solved the crime with the help of " + sidekick +"!");
       },
       addToNotebook: function(newNote){
         notebook.push(newNote);
       } 
     }
   };

If you wanted, you could also use the object notation that employs the this keyword:

   function Detective(name, sidekick){
     var notebook = []; 
     notebook.push("It was a dark and stormy night"); 
     this.getName = function() {	
       return name;
     };	
     this.getSidekick = function(){
       return sidekick;
     };		
     this.setName = function(newName) {
       name = newName;	
     };
     this.setSidekick = function (newSidekick){
       sidekick = newSidekick;
     };
     this.solveCrime = function(){ 
       console.log(name + " has solved the crime with the help of " + sidekick +"!");
     };
     this.addToNotebook = function(newNote){
       notebook.push(newNote);
     };
   };

The closures in this case still consist of each of the member functions along with the local variables that make up their environment (name, sidekick, notebook). Notice that we don't necessarily have to declare separate local variables, either! The arguments passed to the function can in fact serve this purpose nicely.

You can test this out with the following code: 

   var detective1 = new Detective("Sherlock Holmes", "Dr. Watson");
   detective1.solveCrime();
   detective1.setName("Hercule Poirot");
   detective1.solveCrime();

It looks like we've cracked the case! But before we rush to the conclusion that this will work just like, for example, C++ classes, let's move on to examine a few differences, and the problems they might create.
