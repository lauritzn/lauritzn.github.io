Member Functions

The idea of a function that can be called on any object of a specific class is one that will be familiar to anyone with a background in a language like C++.

Before we examine what this looks like in JavaScript, let's talk about what it doesn't look like.

If you create the function as a member of the instance of the class, it will work for that specific object, but not for any other objects of the class. Here's a concrete example, building off the Detective class we just created:

	var detective1 = new Detective("Sherlock Holmes", "Dr. Watson");
	detective1.solveCrime = function(){
		console.log(this.name + " has solved the crime with the help of " + this.sidekick +"!");
	}

If you then call

	detective1.solveCrime();

the following will be logged to the console:

Sherlock Holmes has solved the crime with the help of Dr. Watson!

But if you create a second detective and call the same function, like so: 

	var detective2 = new Detective("Hercule Poirot", "Captain Hastings"); 
	detective2.solveCrime(); // This does not work!!!

The code will break with an error that reads "Uncaught TypeError." This is because there is no function solveCrime() for detective2, only for detective1.

At the same time, it won't work to assign the function directly to the class name:

	Detective.solveCrime = function(){ // This does not work!!
		console.log(this.name + " has solved the crime with the help of " + this.sidekick +"!");
	}

What we need is here is the prototype keyword:

	Detective.prototype.solveCrime = function(){
		console.log(this.name + " has solved the crime with the help of " + this.sidekick +"!");
	}

By adding this function to the prototype for the class, we are building it into the "mold" for the class, from which all the instances of the class are created. Hence all individual Detectives will have this function.

What does this give us so far, and how close are we to C++-style classes? As it turns out, we still have quite a way to go...
